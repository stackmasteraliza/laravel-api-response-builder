# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 3.9.x   | ✅        |
| < 3.0   | ❌        |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability within Laravel API Toolkit, please follow these steps:

1. **Do NOT** disclose the vulnerability publicly until it has been addressed.

2. **Email** the maintainer directly at stackmasteraliza@gmail.com or alizaalidev@gmail.com with:
   - A description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact
   - Any suggested fixes (optional)

3. **Response Time**: You can expect an initial response within 48 hours. We will work with you to understand and address the issue promptly.

4. **Disclosure**: Once the vulnerability is fixed, we will:
   - Release a security patch
   - Credit you in the release notes (unless you prefer to remain anonymous)
   - Publish a security advisory if appropriate

## Security Best Practices

When using this package:

- Always use the latest stable version
- Keep your Laravel installation up to date
- Follow Laravel's security best practices
- Validate and sanitize all user input before passing to API responses
- Use HTTPS in production environments

Thank you for helping keep Laravel API Toolkit secure!
